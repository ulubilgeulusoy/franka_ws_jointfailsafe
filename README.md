# franka_ws_jointfailsafe

Custom ROS 2 Humble workspace for FR3 kinesthetic-teaching backend experiments.

This repository is a source-based Franka workspace snapshot built around a custom joint-limit-aware gravity controller for kinesthetic teaching(https://github.com/ulubilgeulusoy/franka_kinesthetic_teaching_GUI/tree/Humble_KT_failsafe). 

It packages the backend changes needed to support the `Humble_KT_failsafe` kinesthetic-teaching flow in a single workspace repo.

## What This Repository Is

This repository is a ROS 2 `colcon` workspace.

Its top-level structure follows the standard ROS 2 workspace pattern:

```text
franka_ws_jointfailsafe/
  src/
  build/
  install/
  log/
```

Meaning:

- `src/`
  - source code for the ROS packages in this workspace
- `build/`
  - intermediate build output generated by `colcon`
- `install/`
  - installed ROS environment generated by `colcon`
- `log/`
  - build logs generated by `colcon`

Only `src/` and the repository-level documentation/config files should be treated as source-of-truth content. `build/`, `install/`, and `log/` are generated.

## Purpose

The purpose of this workspace is to provide a custom FR3 backend for kinesthetic teaching in which joint-limit behavior is handled more gracefully than in a stock setup.

The core goal is:

- preserve hand-guiding behavior in free motion
- detect approach to configured joint-limit zones
- apply a soft, bounded resisting behavior near those limits
- reduce the chance that users drive the arm directly into hard joint-limit failure behavior during teaching

This repository is therefore focused on backend controller behavior, bringup wiring, and the source dependencies needed to build that backend.

## What Was Changed In This Workspace

This workspace differs from a standard Franka ROS 2 Humble workspace in two main ways:

1. A custom package was added:
   - `src/franka_joint_limit_failsafe`

2. The workspace's Franka ROS 2 source tree was wired to use the custom controller for kinesthetic-teaching gravity mode:
   - `src/franka_ros2/franka_bringup/config/controllers.yaml`

The custom package provides:

- `franka_joint_limit_failsafe/SoftStopGravityController`

This controller is the main custom backend addition in the repository.

## What The Custom Controller Does

`SoftStopGravityController` is an effort-mode gravity-style controller designed for kinesthetic teaching.

Its intended behavior is:

- remain close to gravity compensation away from joint limits
- activate only near configured inner soft-limit regions
- add damping and inward resisting torque as the arm approaches those regions
- keep the wall effect bounded rather than producing an unbounded or abrupt reaction

In practical terms, it is meant to make a joint limit feel more like a soft wall during teaching instead of allowing the user to push directly into a problematic limit condition.

This makes the workspace easier to:

- version as one backend snapshot

## Controller Wiring

The custom controller is wired through:

- `src/franka_ros2/franka_bringup/config/controllers.yaml`

The intended kinesthetic-teaching gravity path in this workspace uses:

- `soft_stop_gravity_controller`

instead of the stock gravity-compensation example controller.

So the key backend change in this repository is not just the addition of a new package, but also the integration of that package into the workspace's controller configuration.

## How `colcon build` Applies Here

This repository is built with `colcon`.

When `colcon build` is run at the repository root, it:

1. scans `src/` for ROS packages
2. resolves dependency order
3. builds the required packages
4. writes intermediate artifacts into `build/`
5. installs the resulting ROS environment into `install/`

After a successful build, the workspace is used by sourcing:

```bash
source install/setup.bash
```

That environment is what exposes the packages, libraries, launch resources, and controller plugins built from this repository.

## When Rebuilds Are Needed

You typically need to rebuild this workspace when you change backend ROS code inside this repository.

Common cases include:

- changes under `src/franka_joint_limit_failsafe`
- changes under `src/franka_ros2`
- changes under `src/libfranka`
- changes to package manifests
- changes to `CMakeLists.txt`
- changes to controller config that must be installed into the workspace environment

Typical commands are:

```bash
colcon build
```

or package-limited builds such as:

```bash
colcon build --packages-select franka_joint_limit_failsafe
colcon build --packages-select franka_bringup
```

## Tuning Location

The main tuning for the soft-stop controller lives in:

- `src/franka_ros2/franka_bringup/config/controllers.yaml`

Key parameters include values such as:

- `soft_limit_margin`
- `activation_zone_width`
- `velocity_damping`
- `limit_stiffness`
- `inward_damping`
- `max_limit_torque`

These parameters shape where the soft wall begins, how strongly it reacts, and how intrusive it feels during hand-guiding.
