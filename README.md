# franka_ws_jointfailsafe

Custom ROS 2 Humble workspace for the FR3 kinesthetic-teaching backend.

This workspace is intentionally separate from the standard baseline workspace at [`/home/parc/franka_ws`](/home/parc/franka_ws). It is the custom backend used by the standalone kinesthetic-teaching GUI in [`/home/parc/franka_kinesthetic_teaching_GUI`](/home/parc/franka_kinesthetic_teaching_GUI).

The purpose of this workspace is to provide an isolated place for:

- kinesthetic teaching backend changes
- controller experiments
- bringup/config wiring changes
- FR3 teach/gravity mode behavior tuning

It is not the standalone GUI repo, and it is not the visual-servo project.

## What This Workspace Is

This is a normal ROS 2 `colcon` workspace.

Its structure follows the standard pattern:

```text
franka_ws_jointfailsafe/
  src/
  build/
  install/
  log/
```

Meaning:

- `src/`
  - source code for ROS packages
- `build/`
  - intermediate build output generated by `colcon`
- `install/`
  - the built ROS environment used after sourcing `install/setup.bash`
- `log/`
  - build logs generated by `colcon`

Only `src/` and project-level docs/config are source-of-truth content. `build/`, `install/`, and `log/` are generated.

## How `colcon build` Applies Here

When you run `colcon build` in this workspace, it:

1. scans `src/` for ROS packages
2. resolves their dependency order
3. builds the required packages
4. writes intermediate output into `build/`
5. installs the usable ROS environment into `install/`

This workspace is then used by sourcing:

```bash
source /home/parc/franka_ws_jointfailsafe/install/setup.bash
```

That environment is what lets ROS find packages such as:

- `franka_bringup`
- `franka_gripper`
- `franka_description`
- `franka_hardware`
- `franka_fr3_moveit_config`
- `franka_joint_limit_failsafe`

## When You Need To Rebuild This Workspace

You usually rebuild this workspace when you change backend ROS code inside it.

Typical examples:

- changes in `src/franka_joint_limit_failsafe`
- changes in `src/franka_ros2`
- changes in `src/libfranka`
- changes in package manifests or `CMakeLists.txt`
- changes in bringup/config that must be installed into the ROS environment

Typical build commands are:

```bash
colcon build
```

or package-limited builds such as:

```bash
colcon build --packages-select franka_joint_limit_failsafe
colcon build --packages-select franka_bringup
```

## Why This Workspace Should Exist Separately

Keeping this workspace separate from `franka_ws` is useful because it allows:

- controller experimentation without modifying the baseline workspace
- safer iteration on teach/gravity behavior
- a reproducible backend snapshot for lab use
- branch-based experimentation for future controller ideas

This is the main reason this workspace is a good candidate for its own GitHub repository.

## GitHub Packaging Intent
